The waiting time calculation was also a little difficult because I needed to track time properly and make sure the results were accurate.

**Most interesting learning**: 
The most interesting part was adding the new features one by one and seeing how each one improved the program. I learned how to add process priority, count context switches, and track waiting time in a simulation. I also understood better how these features make the scheduler output more complete and useful. 
What I added to the code:
I added a priority field ( Feature 1 ) to each process and made the program generate random priorities from 1 to 5. I also added a context switch counter ( Feature 2 )  so the simulation could count how many times the CPU changed from one process to another. In addition, I added waiting time tracking ( Feature 3 ) by storing creation time and calculating how long each process stayed in the system before running. At the end, I created a summary table that shows the process name, burst time, and waiting time.
**What I would do differently next time**: 
Next time, I would start testing each feature earlier and more often. I would also organize the changes more carefully so it is easier to commit each feature separately and keep track of my progress.
